DATABASE SCHEMA
===============

subscribers (synced from MyASP)
  id: UUID (PRIMARY KEY)
  email: TEXT (UNIQUE)
  name: TEXT
  status: TEXT ('active' | 'cancelled')
  myasp_data: JSONB
  created_at: TIMESTAMPTZ
  updated_at: TIMESTAMPTZ
  INDEX: email

otp_codes (for authentication)
  id: UUID (PRIMARY KEY)
  email: TEXT
  code: TEXT (6 digits)
  expires_at: TIMESTAMPTZ (10 min validity)
  used: BOOLEAN
  created_at: TIMESTAMPTZ
  INDEX: email

chat_threads (conversation threads)
  id: UUID (PRIMARY KEY)
  user_email: TEXT (FOREIGN KEY → subscribers.email)
  title: TEXT ('新しいチャット')
  created_at: TIMESTAMPTZ
  updated_at: TIMESTAMPTZ
  INDEX: user_email

chat_messages (message history)
  id: UUID (PRIMARY KEY)
  thread_id: UUID (FOREIGN KEY → chat_threads.id)
  role: TEXT ('user' | 'assistant')
  content: TEXT
  created_at: TIMESTAMPTZ
  INDEX: thread_id

API ENDPOINTS
=============

Authentication:
  POST   /api/auth/send-otp          → Send OTP email
  POST   /api/auth/verify-otp        → Verify code & create session
  POST   /api/auth/logout            → Clear session

Chat:
  POST   /api/chat                   → Stream AI response

Threads:
  GET    /api/threads                → List user threads
  POST   /api/threads                → Create new thread
  PATCH  /api/threads                → Update thread title
  GET    /api/threads/[id]           → Get thread with messages
  DELETE /api/threads/[id]           → Delete thread

Messages:
  GET    /api/messages/[threadId]    → Get messages for thread

Webhooks:
  POST   /api/webhook/myasp          → MyASP webhook (secured)

Admin:
  POST   /api/admin/refresh-content  → Clear transcript cache

==========
